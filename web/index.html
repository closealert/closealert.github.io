Cookies

CloseAlert plaatst automatisch cookies op de computer van de bezoeker om het navigatiepad op uw site bij te houden. Dankzij deze cookies kunt u in het dashboard terugzien op welke manier een bezoeker door uw site is gelopen voordat hij/zij feedback gaf.

Mocht u deze cookies willen uitschakelen, dan is dit mogelijk door de volgende regel toe te voegen aan de CloseAlert code:

__clslrt.setting("notrack", true);

Customizen

De code voor CloseAlert voor websites is geschreven om maximale flexibiliteit te geven aan de webmaster. Hierdoor kunt u zowel de tekst als het uiterlijk geheel aanpassen aan uw organisatie.

Tekst aanpassen

Om de tekst aan te passen kunt u gebruikmaken van de functie __clslrt.locale. Deze functie heeft de volgene syntax: 


__clslrt.locale("<TAAL_ID>", {
  <BERICHT_ID>: "<Tekst voor BERICHT_ID>",
  <TEKST_ID>: "<Tekst voor TEKST_ID>"
});
Hierin zijn de volgende waarden belangrijk:
  • <TAAL_ID> Het tweeletterige id van de taal, bijv. nl voor Nederlands
  • <TEKST_ID> Het id van de tekst, bijv. widgetTitleMobile voor de tekst die in de mobiele weergave naast de knoppen staat of messagePlaceholderPos voor de tekst die bij positieve feedback in het tekstvak staat.

De volledige taalconfiguratie voor Nederlands is als volgt: 


__clslrt.locale('nl', {
    titlePos: 'positieve feedback',
    titleNeg: 'negatieve feedback',

    messagePlaceholderPos: 'vertel even kort waarom je positief bent over deze pagina',
    messagePlaceholderNeg: 'vertel even kort waarom je negatief bent over deze pagina',

    widgetTitle: 'Feedback',
    widgetTitleMobile: 'Feedback',

    emailPlaceholder: 'e-mailadres...',

    send: 'stuur',

    sending: 'Bezig met versturen...',
    sent: 'we hebben je feedback ontvangen!',

    messageTooShort: 'je bericht is te kort',
    emailIncorrect: 'je e-mailadres is niet juist',
    and: 'en'
});

Knoppen aanpassen

Om de iconen in de knoppen aan te passsen dient u delen uit de CSS die door CloseAlert wordt gegenereerd aan te passen. Let op: Hiervoor is goede kennis van HTML en CSS nodig.

Voor de knoppen wordt per browser een andere afbeelding geladen, met andere afmetingen. Dit is om de knoppen op alle ondersteunde browsers optimaal weer te geven. Hieronder staan de specificaties van de afbeelding per browser. U kunt ook een voorbeeld bekijken om de exacte positionering van de verschillende elementen af te leiden.

Internet explorer 8 (Voorbeeld)
In IE8 worden de knoppen volledig in de sprite voor de CloseAlert widget meegenomen. Dit betekent dat naast de knoppen er ook andere elementen in de sprite zitten.
Type: Sprite van 26 x 110 pixels (wxh) met transparante achtergrond.
CSS:


.__clslrt_msie_8 .__clslrt_widget .__clslrt_button {
  background-image: url('URL_NAAR_AFBEELDING_VOOR_IE8') !important;
}

Internet explorer 7 (Voorbeeld)
In IE7 zitten de knoppen in een niet-transparente sprite omdat deze browser transparantie in PNG-afbeeldingen nog niet correct ondersteunt. Daarnaast zitten er geen andere elementen in de sprite.
Type: Sprite van 26 x 56 pixels (wxh) met witte achtergrond.
CSS:


.__clslrt_msie_7 .__clslrt_widget .__clslrt_button {
  background-image: url('URL_NAAR_AFBEELDING_VOOR_IE7') !important;
}

Andere browsers (Voorbeeld)
In andere browsers worden de iconen over de (gekleurde) achtergrond van de knoppen heen geleged. Ook wordt in andere browsers rekening gehouden met schermen van een hogere resolutie (zoals bij tablets dat het geval is). Om deze reden is de sprite in andere browsers van een hogere resolutie.
Type: Sprite van 48 x 200 pixels (wxh) met transparante achtergrond.
CSS:


.__clslrt_widget_thumbs .__clslrt_button {
    background-image: url('URL_NAAR_AFBEELDING') !important;
}

Mobiele (telefoon) browsers (Voorbeeld)
Ook in mobiele browsers worden de iconen over de (gekleurde) achtergrond van de knoppen heen geleged. Aangezien steeds meer smartphones retina-schermen krijgen is de sprite in een nog hogere resolutie.
Type: Sprite van 128 x 500 pixels (wxh) met transparante achtergrond.
CSS:

C
.__clslrt_mobile .__clslrt_widget .__clslrt_widget_faces .__clslrt_button {
    background-image:url('URL_NAAR_AFBEELDING_VOOR_MOBIEL') !important;
}
